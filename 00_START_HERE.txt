QUICK START (2 minutes)
================================================================================

Install dependencies:
  pip install -r requirements.txt

Run your first backtest:
  python scripts/run_backtest.py --station NYC --start 2025-01-01 --end 2025-06-30

This will:
  1. Download METAR observations from Iowa State
  2. Engineer features at 1 PM UTC
  3. Run walk-forward XGBoost backtest
  4. Print accuracy metrics
  5. Generate 3 plots in outputs/

Run tests:
  pytest tests/ -v

================================================================================
INTERVIEW TALKING POINTS
================================================================================

1. "Walk-forward backtesting prevents look-ahead bias"
   Standard train/test split allows the test set to influence model selection.
   Walk-forward training only on data before the test date accurately reflects
   real-world deployment.

2. "I integrated multiple APIs with transparent caching"
   Fetching from Iowa State IEM and Open-Meteo with CSV/JSON caching to avoid
   redundant downloads.

3. "Meteorologically informed features"
   Features based on weather science (diurnal cycles, local heating effects,
   temperature trends) not just ML pattern matching.

4. "Production-grade code quality"
   Type hints, docstrings, error handling, logging, tests - this is code
   suitable for deployment.

5. "Proper temporal validation for time series"
   Understanding that time-series models need special validation to prevent
   information leakage from the future.
