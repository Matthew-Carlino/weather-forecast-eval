================================================================================
INTERVIEW TALKING POINTS
================================================================================

1. WALK-FORWARD BACKTESTING
   "I implemented proper temporal validation with walk-forward backtesting.
    Each day, the model trains only on data strictly before that date,
    preventing look-ahead bias and simulating realistic deployment."

2. API INTEGRATION & CACHING
   "I integrated two external APIs with transparent caching:
    - Iowa State IEM for historical METAR observations
    - Open-Meteo for numerical weather forecasts
    The caching strategy prevents redundant downloads while allowing
    force_refresh for data updates."

3. METEOROLOGICALLY INFORMED FEATURES
   "The features are based on weather science, not just ML:
    - Running daily max captures temperature momentum
    - Airport spread detects local heating effects
    - Diurnal progress captures time-of-day cycles
    - Trend captures rate of change"

4. PRODUCTION CODE QUALITY
   "100% type-hinted functions, Google-style docstrings, comprehensive
    error handling, proper logging, unit tests with pytest, pure functions
    for testability. This is production-grade code suitable for deployment."

5. TEMPORAL VALIDATION IMPORTANCE
   "Walk-forward backtesting is critical for time-series forecasting.
    Using standard train/test split causes look-ahead bias because test data
    affects model selection. Walk-forward prevents this and accurately
    reflects real-world performance."
