================================================================================
KEY FEATURES
================================================================================

1. DATA FETCHING
   • METARFetcher: Hourly observations from Iowa State University
   • ForecastFetcher: Model forecasts from Open-Meteo (ICON, GFS, ECMWF)
   • Automatic CSV/JSON caching with date-range keys
   • Timezone-aware (all UTC internally)

2. FEATURE ENGINEERING
   • Pure functions for composability and testability
   • 9 core features: running_max, deltas, trends, diurnal_progress, spreads
   • Multi-station support for airport comparisons
   • Monthly-adjusted expected diurnal ranges
   • NaN-free output (forward/backward fill + zeros)

3. MACHINE LEARNING
   • XGBoost classifier (150 trees, max_depth=4)
   • Walk-forward backtesting (expanding window, no future peeking)
   • Periodic retraining (configurable every N days)
   • Confidence scores from predicted probabilities
   • Feature importance via model.gain

4. VISUALIZATION
   • 6 plotting functions (all publication-ready at 150 DPI)
   • Accuracy by month/time, feature importance, confusion matrix
   • Calibration curves, confidence distribution
   • Optional save_path for automated report generation
   • Consistent color scheme (seaborn husl palette)

================================================================================
DESIGN PATTERNS DEMONSTRATED
================================================================================

1. Pure Functions
   - All feature engineering functions are stateless
   - Easy to test, parallelize, and understand

2. Proper Temporal Validation
   - Walk-forward prevents look-ahead bias
   - Never trains on test data
   - Simulates realistic deployment

3. Type System
   - Full type hints on all signatures
   - Better IDE support and documentation

4. Configuration Management
   - Constants at module top (XGB_PARAMS, IEM_STATION_MAP, etc.)
   - No magic numbers in code
   - Easy to tune and experiment

5. Error Handling
   - Try/except with informative messages
   - Graceful fallbacks (e.g., empty airport list)
   - Logging of warnings and info

6. Documentation
   - Module-level docstrings
   - Function docstrings (purpose, args, returns, raises)
   - Usage examples in README and QUICKSTART
